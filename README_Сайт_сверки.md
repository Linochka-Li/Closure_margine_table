# Сайт для закрытия подрядчиков — как пользоваться и что важно знать

## Что это

Один файл `index.html` — работает полностью в браузере, ничего никуда не отправляет
(вся обработка — на компьютере пользователя). Внутри та же логика, что была в
Python-версии (`processor.py`/`updater.py`), проверенная построчно на ваших реальных
файлах (Sputnik24, Buzzoola, MoeVideo) — суммы совпадают до копейки, включая
последнее правило (цена/бюджет первичны для границы Adfox/DSP).

## Как запустить

Просто откройте `index.html` двойным щелчком — работает в любом браузере без
установки. Чтобы коллеги могли пользоваться по ссылке (а не пересылать файл),
можно захостить его на любом статическом хостинге — это один файл без бэкенда:
- Google Sites / Google Drive (расшарить как веб-страницу)
- GitHub Pages, Netlify, Vercel (просто перетащить файл)
- Внутренний файловый сервер компании

## Как пользоваться (для коллег)

1. **Шаг 1** — загрузить .xlsx с текущим листом месяца из таблицы маржинальности
   (Файл → Скачать → Microsoft Excel в гугл-таблице). Сайт сам найдёт подрядчиков и
   список проектов.
2. **Шаг 2** — загрузить файл сверки (от подрядчика, DSP-биллинг, МТС или Директ).
   Тип определяется автоматически.
3. **Шаг 3** — точные и уверенные совпадения применяются сами. Здесь показывается
   только то, что нужно подтвердить: несколько похожих проектов или новое название,
   которого раньше не встречалось. Достаточно нажать на нужный вариант.
4. **Шаг 4** — сводка по подрядчикам + таблица всех изменений с точным адресом
   ячейки (например, `AN15`). Кнопка «Скачать .xlsx» — выгружает эту таблицу.
   Можно вернуться на шаг 2 и загрузить следующий файл — изменения накапливаются
   в рамках одной сессии (пока не закрыта вкладка).

## Важное ограничение — и почему так

Сайт **не переписывает саму гугл-таблицу** и не создаёт «обновлённую копию» всей
таблицы маржинальности целиком. Он выгружает **таблицу изменений**: строка,
проект, подрядчик, блок (Adfox/DSP), точный адрес ячеек, показы и сумма.

Это осознанное решение, а не недоделка: библиотека для чтения/записи Excel в
браузере (SheetJS, бесплатная версия) не умеет надёжно сохранять всё форматирование,
объединённые ячейки и формулы в сложных многолистовых книгах вроде вашей таблицы
маржинальности. Если бы сайт пересобирал весь файл целиком, был бы риск случайно
испортить оформление остальных листов (Февраль24 … Март25 и т.д.), а не просто
дописать новые цифры.

Поэтому финальный шаг — быстро скопировать значения из выгруженного файла в
нужные ячейки гугл-таблицы (или, если нужно полностью автоматизировать именно
запись, — воспользоваться Apps Script-версией, которую мы делали раньше: она
пишет прямо в гугл-таблицу через официальный API, минуя это ограничение).

## Кэш подсказок («кампания → проект»)

Артефакты Claude запрещают использовать `localStorage`/`sessionStorage` — поэтому
кэш подтверждённых сопоставлений хранится **в памяти вкладки браузера**, пока она
открыта. Кнопки «Экспорт» / «Импорт» в левой панели позволяют:
- сохранить накопленные подсказки в `mapping_cache.json`,
- загрузить чей-то файл с подсказками в начале новой сессии.

Практика для команды: раз в неделю кто-то один экспортирует свежий файл подсказок
и выкладывает его в общую папку — остальные подгружают его в начале работы, чтобы
не сопоставлять одни и те же кампании заново.

## Выбор листа месяца

Каждый месяц в файле таблицы маржинальности появляется новый лист
(`June_26`, `July_26`, `Июнь26`... — оба варианта написания распознаются).
На шаге 1 сайт сам предлагает нужный лист (по текущему месяцу и году), но
выбор всегда можно поменять вручную в выпадающем списке.

Выбранный лист виден **на каждом шаге** — плашка «Лист месяца» в левой панели.
Клик по ней в любой момент возвращает на шаг 1, чтобы сменить лист.

Если переключить лист месяца, когда уже есть подтверждённые или ожидающие
подтверждения строки — сайт предупредит и попросит подтвердить, что текущий
прогресс нужно очистить (номера строк относятся к конкретному листу, при
смене месяца становятся неверными, поэтому автоматически перенести их нельзя).

## Настройки (шаг 1, «Расширенные настройки листа»)

Если в будущем поменяется структура таблицы (другая строка заголовков, другая
колонка проекта) — это настраивается здесь же, без правки кода.

Пороги нечёткого сопоставления и список SSP→DSP-колонка зашиты в начале
JS-кода (`STATE.settings` в файле) — те же значения, что были в `config.py`.
Расширять список SSP при появлении новых DSP-партнёров можно прямо там.

## Что перенесено 1-в-1 из Python-версии (проверено)

- Удаление `olv_campaign` и полностью пустых строк.
- Схлопывание повторяющихся кампаний.
- Граница Adfox/DSP — первая дробная цена ИЛИ первый скачок бюджета с меньшего
  на большее значение (что раньше), это же правило работает единообразно для
  ВСЕХ подрядчиков.
- DSP-строки без названия кампании — по умолчанию исключаются автоматически,
  но это настраивается тумблером прямо на шаге 2 («Автоматически исключать DSP
  без названия кампании») — если выключить, они уйдут в очередь на подтверждение,
  как раньше в Python-версии до последнего уточнения.
- Нечёткое сопоставление кампания↔проект: тот же вес-формула, то же исправление
  токенизации (CamelCase, дефис между буквой и цифрой), та же защита от
  автосклейки разных месяцев.
- Поиск колонок подрядчика в таблице — тот же исправленный алгоритм (сканирует
  всю строку заголовков, а не останавливается на первом совпадении).

Все эти пункты сверены построчно через `node` на ваших реальных файлах перед
сдачей — см. `logic.js` + `test_logic.js` в комплекте, если понадобится
перепроверить или расширить логику в будущем.
